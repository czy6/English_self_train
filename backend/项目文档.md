## 📋 项目概述

英语智能训练平台是一个基于微服务架构的英语学习与训练系统，整合了 Spring AI、LangChain4j、XGBoost 等先进技术，为用户提供智能化的英语学习、训练、评测一体化解决方案。

### 核心功能

- 🎯 **个性化训练**：基于 XGBoost 模型的熟练度预测与题目推荐
- 🤖 **AI 智能批改**：翻译/写作题目 AI 自动批改与评分
- 💬 **智能对话**：Spring AI + LangChain4j 双引擎 AI 对话（支持 SSE 流式输出）
- 📊 **学情分析**：薄弱知识点分析、学习轨迹追踪
- 📚 **题库管理**：多级分类、难度分级、智能组卷
- 🔐 **安全认证**：JWT + RSA256 签名、网关统一鉴权

#### 1.4 用户信息工具 (UserHolder)

- **技术**: `ThreadLocal<Long>`
- **功能**: 存储和获取当前登录用户 ID

```java
public class UserHolder {
    private static final ThreadLocal<Long> tl = new ThreadLocal<>();

    public static void setUserId(Long userId) {
        tl.set(userId);
    }

    public static Long getUserId() {
        return tl.get();
    }

    public static void removeUserId() {
        tl.remove();
    }
}
```

**⚠️ 注意事项**:

- 必须在请求完成后调用 `removeUserId()` 防止内存泄漏
- 仅在请求处理线程内有效，不可跨线程使用
- 异步线程中需要手动传递用户 ID

#### 1.5 用户信息拦截器 (UserInfoInterceptor)

- **技术**: `HandlerInterceptor`
- **功能**: 从请求头提取用户信息并存入 ThreadLocal

**执行流程**:

```
1. 从请求头获取 "user-info"
2. 解析用户 ID
3. 存入 UserHolder
4. 执行目标方法
5. 清理 UserHolder
```

#### 1.6 缓存客户端 (CacheClient)

- **技术**: Redis + 逻辑过期 + 缓存空值 + 互斥锁

**解决缓存三大问题**:

| 问题     | 现象              | 解决方案                |
| -------- | ----------------- | ----------------------- |
| 缓存穿透 | 查询不存在的数据  | 缓存空值（TTL: 2 分钟） |
| 缓存击穿 | 热点 key 过期     | 逻辑过期 + 互斥锁重建   |
| 缓存雪崩 | 大量 key 同时过期 | 随机 TTL（±5 分钟）     |

**核心方法**:

```java
// 缓存穿透解决方案
public <R, ID> R queryWithPassThrough(
    String keyPrefix, ID id, Class<R> type, Function<ID, R> dbFallback, Long time, TimeUnit unit
);

// 缓存击穿解决方案（逻辑过期）
public <R, ID> R queryWithLogicalExpire(
    String keyPrefix, ID id, Class<R> type, Function<ID, R> dbFallback, Long time, TimeUnit unit
```

### 2. english-gateway（API 网关模块）

**技术栈**: Spring Cloud Gateway + Nacos + JWT + Security

#### 2.1 全局认证过滤器 (AuthGlobalFilter)

- **技术**: `GlobalFilter` + `Ordered`
- **功能**: 统一认证鉴权

**执行流程**:

```
1. 判断是否为白名单路径 → 是则放行
2. 从 Authorization 请求头获取 Token
3. 使用 JwtTool 解析 Token
   - Token 无效 → 返回 401
   - Token 过期 → 返回 401
4. 提取用户 ID 存入请求头 "user-info"
5. 放行请求到下游服务
```

**白名单配置**:


#### 2.2 JWT 工具 (JwtTool)

- **技术**: Hutool JWT + RS256 签名

**核心方法**:


```
  "exp": 1714492800     // 过期时间
```

### 3. user-service（用户服务模块）

**技术栈**: Spring Boot + MyBatis-Plus + Redis + JWT + 阿里云 OSS

#### 3.1 用户登录 (UserServiceImpl)

- **技术**: Redis 验证码 + JWT Token
- **流程**:

```
1. 校验手机号格式（11 位数字）
2. 生成 6 位随机验证码
3. 验证码存入 Redis（TTL: 5 分钟）
   Key: "code:" + phone
   Value: 验证码
4. 调用短信服务发送验证码（模拟）
5. 用户提交验证码
6. 从 Redis 获取验证码并比对
   - 不一致 → 返回错误
   - 一致 → 继续
7. 查询用户（根据手机号）
   - 不存在 → 创建用户
8. 生成 JWT Token 返回
```

**关键代码**:

```java
@Override
public Result login(LoginDTO loginDTO) {
    // 1. 校验手机号
    if (!RegexUtils.isPhoneInvalid(loginDTO.getPhone())) {
        return Result.fail("手机号格式错误");
    }

    // 2. 生成验证码
    String code = RandomUtil.randomNumbers(6);

    // 3. 存入 Redis
    stringRedisTemplate.opsForValue().set(
        CODE_KEY + loginDTO.getPhone(),
        code,
        5, TimeUnit.MINUTES
    );

    // 4. 发送短信（模拟）
    log.info("验证码：{}", code);

    return Result.ok("验证码已发送");
}
```

```java
public List<Proficient> selectBaseTypeInfo(String mainType, String subType) {
    // 1. 查询用户所有作答记录
    List<ProficientBaseDTO> baseLists = baseMapper.selectBaseTypeInfo(mainType, userId);

    // 2. 按 (subType, difficulty) 分组
    Map<String, Set<Long>> allMap  = new LinkedHashMap<>();
    Map<String, Set<Long>> doneMap = new LinkedHashMap<>();

    for (ProficientBaseDTO dto : baseLists) {
        String key = dto.getSubType() + "#" + dto.getDifficulty();
        allMap.computeIfAbsent(key, k -> new LinkedHashSet<>()).add(dto.getQuestionId());
        if (dto.getDone() == 1) {
            doneMap.computeIfAbsent(key, k -> new LinkedHashSet<>()).add(dto.getQuestionId());
        }
    }

    // 3. 计算熟练度
    List<Proficient> proficientList = new ArrayList<>();
    for (String key : allMap.keySet()) {
        Set<Long> allIds = allMap.get(key);
        Set<Long> doneIds = doneMap.getOrDefault(key, Collections.emptySet());

        double proficient = calculateProficiency(allIds, doneIds);

        Proficient proficient = new Proficient();
        proficient.setMainType(mainType);
        proficient.setSubType(key.split("#")[0]);
        proficient.setDifficulty(key.split("#")[1]);
        proficient.setProficient(proficient);
        proficientList.add(proficient);
    }

    return proficientList;
}
```

#### 6.2 AI 对话接口（SSE 流式版本）⭐

- **技术**: Server-Sent Events + Reactor Flux
- **功能**: 实现打字机效果的流式输出

**Controller 层**:

```java
@RequestMapping(value = "/chat", produces = MediaType.TEXT_EVENT_STREAM_VALUE)
public SseEmitter chatStream(
        @RequestParam("prompt") String prompt,
        @RequestParam("chatId") String chatId) {

    SseEmitter emitter = new SseEmitter(300000L);

    // 设置回调
    emitter.onCompletion(() -> log.info("SSE 连接完成"));
    emitter.onTimeout(() -> log.warn("SSE 连接超时"));
    emitter.onError(throwable -> log.error("SSE 连接错误", throwable));

    // 异步处理流式响应
    chatService.chatStream(prompt, chatId, emitter);

    return emitter;
}
```

**Service 层**:

```java
@Override
public void chatStream(String prompt, String chatId, SseEmitter emitter) {
    // 1. 修改对话标题（如果是第一次对话）
    modifyTitle(chatId, prompt);

    // 2. 保存用户 ID（异步线程需要手动传递）
    Long userId = UserHolder.getUserId();

    // 3. 异步处理流式响应
    CompletableFuture.runAsync(() -> {
        UserHolder.setUserId(userId);  // 在异步线程中设置用户 ID
        try {
            consultService.chatStream(chatId, prompt)
                .collectList()  // 收集所有 token
                .subscribe(
                    tokens -> {
                        String reply = String.join("", tokens);

                        // 流式发送到前端
                        for (String token : tokens) {
                            emitter.send(SseEmitter.event()
                                    .name("message")
                                    .data(token));
                        }

                        // 发送完成信号
                        emitter.send(SseEmitter.event()
                                .name("done")
                                .data("[DONE]"));
                        emitter.complete();

                        log.info("流式对话完成 - chatId: {}, AI 回复：{}", chatId, reply);
                    },
                    error -> {
                        log.error("Flux 订阅错误", error);
                        emitter.completeWithError(error);
                    }
                );
        } finally {
            UserHolder.removeUserId();  // 清理用户 ID
        }
    }, executor);
}
```

## 🤖 AI 能力详解

### AI 对话双引擎架构

系统提供两套 AI 对话服务，各有优劣：

| 特性         | Spring AI             | LangChain4j                            |
| ------------ | --------------------- | -------------------------------------- |
| 框架成熟度   | Spring 官方，稳定性高 | 社区驱动，功能丰富                     |
| 流式输出     | 支持                  | 支持（SSE）⭐                          |
| 对话记忆     | 需要自己实现          | 内置 ChatMemory ⭐                     |
| Tool 调用    | 支持                  | 支持（更灵活）⭐                       |
| RAG 支持     | 基础支持              | 完善支持（Embedding Store）⭐          |
| 配置复杂度   | 简单                  | 中等                                   |
| 推荐使用场景 | 简单对话场景          | 复杂对话、需要记忆、Tool 调用等场景 ⭐ |

### SSE 流式输出详解

**优势**:

- ✅ 实时性更好，用户体验佳（打字机效果）
- ✅ 减少首屏等待时间
- ✅ 降低服务器缓冲压力

**实现原理**:

```
用户发送消息
    ↓
[主线程] 创建 SseEmitter
    ↓
[异步线程] 调用 LangChain4j AI Service
    ↓
Flux 发射 Token（逐字）
    ↓
逐字发送到前端（SSE event）
    ↓
Flux 完成
    ↓
LangChain4j 自动保存对话记录到 ChatMemory
    ↓
RedisChatMemoryStore.updateMessages()
    ↓
保存到 MySQL + Redis
```

**前端调用示例**:

```javascript
const eventSource = new EventSource("/ai/chat?prompt=你好&chatId=123");

eventSource.onmessage = (event) => {
  // 逐字显示
  displayText += event.data;
};

eventSource.addEventListener("done", () => {
  console.log("对话完成");
  eventSource.close();
});

eventSource.onerror = (error) => {
  console.error("SSE 错误", error);
  eventSource.close();
};
```

## 🔐 安全机制

### 1. 认证流程

```
1. 用户登录 → 获取 Token
2. 后续请求携带 Token（Authorization: Bearer {token}）
3. Gateway 统一鉴权
   - Token 无效 → 返回 401
   - Token 有效 → 提取用户 ID → 放行
4. 下游服务从请求头获取用户 ID
```

### 2. Token 续期机制

**策略**: 剩余 1/3 有效期时自动续期

```java
@GetMapping("/user/refresh")
public Result<String> refreshToken(@RequestHeader("Authorization") String token) {
    if (jwtTool.shouldRenew(token)) {
        String newToken = jwtTool.createToken(userId, 30, TimeUnit.DAYS);
        return Result.ok(newToken);
    }
    return Result.ok(token);
}
```

### 3. 常见问题

#### Q1: Token 无效

**原因**: Token 过期或签名不匹配

**解决**:

1. 检查 Token 是否过期
2. 检查 JKS 密钥文件是否正确
3. 重新登录获取新 Token

#### Q2: 用户 ID 为 null

**原因**: UserHolder 中用户 ID 未设置

**解决**:

1. 检查请求是否经过 Gateway 认证
2. 检查请求头是否包含 "user-info"
3. 检查拦截器配置

#### Q3: 流式对话没有 AI 回复

**原因**: 模型配额用完或 API Key 无效

**解决**:

1. 检查 API Key 是否正确
2. 检查阿里云百炼控制台配额
3. 查看日志中的详细错误信息
